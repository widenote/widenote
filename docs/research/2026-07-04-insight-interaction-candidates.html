重度洞察技术方案

建议采用“先证据，后观点”的流水线。它像一个中立观察者，但每一句判断都能回到来源。

1. Pattern Indexer 捕获、Memory、卡片变化后，写轻量 pattern candidates。只记录结构和来源，不下大结论。
2. Insight Scout 按 7/14/30 天窗口和 lens 找候选主题。需要 semantic search 和 activity stats，不能靠关键词。
3. Synthesizer 用中立人格生成一条 insight：主张、为什么重要、证据、反向证据、开放问题、小实验。
4. Source Auditor 检查每个 claim 的 source refs、证据数量、跨度、敏感风险、重复度和置信度。
5. Maintainer 根据用户反馈和新证据做置顶、再生成、supersede 或 dismiss，不做无语义的归档。

MemeX 参考转译

  • 保留“洞察是持久对象”，不是聊天里的一次回答。
  • 保留“每条洞察有多个 related facts/source refs”。
  • 保留“洞察可以进入对话，被追问、纠正、扩展”。
  • 不要照搬 P.A.R.A 或文件工作区为核心真相；WideNote 仍以 captures、Memory、SQLite object truth 和 public schema 为中心。

推荐决策

  • 第一阶段选 A 作为主体验：观察者周报最符合“最近状态总结和启发”。
  • 把 C 的“追问这个洞察”合入详情页，作为 Aha Moment 的延伸。
  • B 的 Activity Stats 可以作为第二个分段页，不急着首发。
  • D 等生成质量稳定后再做，否则卡片越漂亮，空泛感越明显。
