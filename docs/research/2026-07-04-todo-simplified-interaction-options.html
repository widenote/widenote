WideNote Todo Redesign, 2026-07-04

待办页和待办卡详情页的简化交互方案

这版重做刻意避开“任务管理器”方向：不把优先级、截止日、缩进、排序、任务信息区放到默认详情里。 MemeX 的参考重点放在日程简报、轻量完成动作、source card、底部动作条和详情页信息密度上。

行动项可以一键完成 日程候选只展示和回源 保留来源，不暴露调参感 详情页只做确认、回源、说明 高级 metadata 默认折叠
建议先决策交互模型 如果要最像 MemeX，选 A。要最快变轻，选 D。要强化可信度，选 C。 B 更像“把 AI 建议扫一遍”的收件箱，适合 Todo agent 还需要用户复核时。
Option A
A

今日焦点流

最重度参考 MemeX schedule：用一个焦点卡加时间流替代多个桶和摘要卡。 详情页是一个清爽 child page，只展示动作、来源和生成理由。

保留 今日焦点、行动和日程分流、完成/恢复、打开来源。
删除 任务信息区、priority/due/indent/sort 设置、搜索默认常驻。
风险 如果待办很多，焦点流需要一个“全部”入口承接。

Option C
