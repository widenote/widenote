# WideNote Workflows

Status: active

This directory describes repeatable human and Codex workflows. It is for
process contracts that are broader than one module but too task-shaped for
`AGENTS.md`, architecture contracts, ADRs, or RFCs.

Use this workflow when a task will change code or prepare a PR:

- [Code Change Orchestration](./code-change-orchestration.md)

Ordinary read-only research, code explanation, and product discussion keep the
normal lightweight context flow unless they become a code-change task. Keep
workflow docs operational and short. Put durable product or architecture
decisions in ADRs/RFCs, then link those decisions from workflow docs when they
change how work should be done.
