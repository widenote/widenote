# WideNote Skills

WideNote repo-scoped Codex skills live here. Each skill is a small directory
with a `SKILL.md` file and should point to workflow docs instead of duplicating
long policy text.

## Code-Change Skill

- `widenote-code-change-orchestrator`: use for new requirements, bug fixes,
  code changes, and PR preparation. It coordinates isolated subagents for
  design, implementation, QA, review, and final human-review evidence.

Ordinary read-only research, code explanation, and product discussion do not
need a repo skill unless they become a code-change task.
